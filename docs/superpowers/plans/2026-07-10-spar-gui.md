# spar GUI (PySide6 dashboard-pilot) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `spar gui` — a native desktop dashboard-pilot (mockup variant A): live stream with per-side colors and filters, task board, dynamic gate panel with one-click decisions (consensus Accept auto-starts exec), run lifecycle controls (new debate / start exec / resume / stop). No embedded terminal — claude/codex stay in the user's terminal.

**Architecture:** One new package `spar/gui/` behind an optional dependency (`pip install spar-cli[gui]` → PySide6). The GUI is the FOURTH consumer of the existing agent-mode protocol — the engine is not modified: it reads `.spar/live.log` (reusing `spar.watch.follow` semantics), polls `spar.status.build_status()` in-process (import, no subprocess), and issues decisions by spawning the same CLI commands an agent would (`spar --continue --headless --quiet --gate …`). Gate buttons render EXCLUSIVELY from `pending_gate.options`, so an illegal decision is unrepresentable. Process ownership: the GUI is a solo pilot — it spawns and owns spar processes via `QProcess` (Stop = SIGINT → exit 130); coexistence with a driving agent is explicitly out of scope for v1 (documented; the GUI shows a read-only warning when it detects a running spar it did not start — lock held → "obserwuję tylko").

**Design tokens (from the accepted mockup):** app is committed-dark. Ground `#14171c`, panels `#1b1f26`/`#20252e`, line `#2c323c`, text `#d5dae2`/muted `#7d8590`; claude `#e8a854`, codex `#56c3d6`, spar-log `#9d8cff`, ok `#6fbf73`, warn `#e0b154`, gate `#e0679a`. Mono for stream (system mono stack), UI sans for chrome. One QSS file holds ALL colors as the single source.

**Tech Stack:** Python 3.11+, PySide6 (optional extra `gui`), pytest + pytest-qt (dev extra), `QT_QPA_PLATFORM=offscreen` for headless tests. Linux/macOS now; **Windows deferred by decision** (core uses fcntl — separate future tranche; noted in HANDOFF backlog).

## Global Constraints

- Engine untouched: NO changes under `spar/` outside `spar/gui/`, `spar/cli.py` (routing only) and `pyproject.toml`. The GUI talks protocol (live.log, build_status, CLI resume commands) — if a needed capability is missing from the protocol, STOP and surface it, don't reach into engine internals.
- Core stays zero-dep: `import spar.gui` may require PySide6, but `spar` CLI without the extra must work exactly as today; `spar gui` without PySide6 prints an install hint and exits 2.
- Gate buttons only from `pending_gate.options`; consensus Accept default = accept + auto `spar exec --headless --quiet` (a secondary "Accept (tylko plan)" skips the auto-exec).
- final_merge NEVER auto-accepts.
- All subprocess spawns use the SAME python/venv (`sys.executable -m spar.cli` style or the resolved `spar` script) and always `--headless --quiet`.
- TDD with pytest-qt for logic-bearing pieces (view-models, gate mapping, runner state machine); pixel-level layout is smoke-checked, not asserted.
- Conventional commits; NO Co-Authored-By / AI-attribution trailers (hard rule).
- Suite green after every task: `python3 -m pytest tests/ -q` (439 passed, 2 skipped before this plan; GUI tests must skip cleanly when PySide6 is absent — `pytest.importorskip("PySide6")` at module top).
- README gets a GUI section in the SAME plan (memory rule: README ships with the feature).

### Task 2: process runner + toolbar (Opus)

**Files:**
- Create: `spar/gui/runner.py` (`SparRunner(QObject)`)
- Create: `spar/gui/toolbar.py` + new-debate dialog
- Test: `tests/test_gui_runner.py`

**Interfaces:**
- `SparRunner` — owns at most ONE `QProcess`; API: `start_debate(task_text, sides, first, tasks: bool)` (writes task_text to a temp task-file), `start_exec()`, `resume(gate_value: str | None)` (debate vs exec chosen from `build_status()['phase']` — `None`/`"debate"` → `spar --continue`, else `spar exec --continue`), `stop()` (SIGINT). Signals: `started(cmd: str)`, `finished(exit_code: int)`, `state_changed(RunnerState)`.
- `RunnerState` enum drives toolbar enablement exactly like the mockup: `IDLE` (Nowa debata on), `RUNNING` (Stop on, rest off), `GATE_PENDING` (all off — decisions live in the gate panel), `RESUMABLE` (Wznów on; entered on exit codes 130/4 and on crash), `DONE` (Nowa debata on). Mapping from `(process alive?, last exit code, build_status().pending_gate/phase)` is a PURE function `derive_state(alive, exit_code, status) -> RunnerState` — unit-tested exhaustively without Qt.
- Auto-exec chain: `resume("accept")` on a CONSENSUS gate → on exit 0, if the auto-exec flag was set (default True, off for "Accept (tylko plan)") → `start_exec()` automatically.
- Lock coexistence: before any spawn, if `.spar/lock` is flock-held by another process (probe: try `spar status` fast path — or simply let the spawn fail with exit 3 and surface the message in the status bar + a read-only banner). v1: surface, don't fight.
- Commands spawned as `[sys.executable, "-m", "spar.cli", ...]`-equivalent; verify `python3 -m spar.cli --help` works (it does via `main`) — if not, fall back to the `spar` console script on PATH.

- [ ] **Step 1: failing tests** — `derive_state` truth table (alive+no gate→RUNNING; not alive+gate→GATE_PENDING; not alive+130→RESUMABLE; not alive+0+phase done→DONE; fresh dir→IDLE). Runner integration with a FAKE spar script (tmp executable printing gate-pending line and exiting 10): start→signals, stop→SIGINT observed, resume builds the right argv (recorded via the fake).
- [ ] **Step 2–3: implement; PASS.**
- [ ] **Step 4: commit** — `feat(gui): SparRunner process pilot + toolbar state machine`

### Task 4: status pane — tasks, gate panel, plan/diff viewers (Sonnet)

**Files:**
- Create: `spar/gui/sidepane.py` (`SidePane`, `TaskBoard`, `GatePanel`), `spar/gui/viewers.py` (plan markdown viewer, diff viewer)
- Test: `tests/test_gui_sidepane.py`

**Interfaces:**
- Status poll: QTimer(2 s) → `spar.status.build_status(project_dir/'.spar')` (in-process import; wrap in try/except — a mid-write state file must not crash the GUI; on StateError keep last good status).
- `TaskBoard` — rows from `status['tasks']` sorted with the engine's numeric task order (`t2` before `t10` — port the tiny key, don't import from exec.loop), pills: merged (ok), review/implementing/testing (warn), pending/ready (muted); right-aligned `side → reviewer` label (reviewer = the other side, mirroring the engine's convention).
- `GatePanel` — hidden when `pending_gate is None`; else renders name-specific header + context (review_rounds: task_id, rounds, open remarks with severity colors; final_merge: summary text incl. diffstat; consensus: artifact link + rounds) and buttons STRICTLY from `options`: `accept` → for consensus TWO buttons ("Accept → start exec" primary with auto-exec flag, "Accept (tylko plan)" secondary), otherwise one Accept; `abort` → Abort (confirm dialog for final_merge abort only? NO — abort is always confirmed with a single yes/no dialog, it ends the run); `extend` → QSpinBox(1..99, default 2) + "Dodaj rundy"; `remarks` → QPlainTextEdit + "Wyślij remarks" (writes tmp file, calls `runner.resume(f"remarks:{path}")`, one remark per line).
- Buttons call `SparRunner.resume(...)`; panel disables itself once clicked until the next status poll (double-click safety).
- `viewers.py`: Plan button → dialog with artifact.md rendered via `QTextBrowser.setMarkdown`; Diff button (enabled when integration branch exists) → `git diff master..spar/integration` in a mono viewer with +/- coloring (subprocess git, read-only).
